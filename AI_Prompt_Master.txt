CORE ARCHITECTURE & GDD - DESKTOP IDLE RPG: TU TIÊN

Tài liệu nội bộ dự án. Trạng thái: Nguyên mẫu (V1.0).
1. Thể loại, Mục tiêu & Vòng lặp cốt lõi

    Thể loại: Desktop Overlay, Idle RPG, Roguelite, Xianxia (Tu tiên).

    Mục tiêu cốt lõi: Trải nghiệm treo máy cày cuốc (Endless Progression) không xâm lấn, chạy nền trực tiếp trên màn hình làm việc của Windows.

    Core Loop: Môi trường cuộn (Khám phá) -> Spawn Quái -> Tiếp cận (Dừng cuộn) -> Auto-Battle (Trừ HP) -> Reset Wave / Xử lý tiến trình -> Lặp lại.

2. Các cơ chế (Mechanics) chính đã chốt

    Desktop Overlay: Cửa sổ game lơ lửng, trong suốt không viền. Cho phép click xuyên thủng (Click-through) ở vùng không có UI.

    Window Management: Kéo thả vùng UI được chỉ định (Ground) để di chuyển game. Hỗ trợ đa màn hình (Virtual Screen) và tự động Clamp chống lọt cửa sổ ra ngoài mép màn hình.

    Responsive UI & Parallax: Giao diện co giãn tự động theo tỷ lệ (Canvas Scaler). Môi trường tạo ảo giác di chuyển bằng cách cuộn các điểm neo thị giác (Grass) sang phải thay vì nhân vật di chuyển.

    Combat Location-Agnostic: Hero neo cố định mép phải, Quái neo ẩn ngoài mép trái. Tính toán khoảng cách va chạm tự động quy đổi theo tỷ lệ chiều rộng thực tế của Canvas.

    Hệ thống Tag-based & Skin: Phân loại sức mạnh bằng mảng chuỗi định danh (VD: Basic_01, Ground_Type). Hiển thị đồ họa (VD: Skin FadeIn_01) chạy Coroutine hoàn toàn độc lập với vòng lặp logic.

3. Kiến trúc Code & Quy chuẩn

    Kiến trúc hiện tại: Phân tách trách nhiệm (Separation of Concerns) kết hợp State Machine cơ bản.

        TransparentWindow.cs: Cô lập toàn bộ giao tiếp cấp hệ điều hành (OS API) và Input.

        UIManager.cs: Cô lập tương tác người dùng (View).

        GameManager.cs: Cô lập logic game, chỉ số và State (Explore/Combat).

    Xử lý Input đột phá: Bỏ qua EventSystem mặc định của Unity. Dò tìm va chạm chuột bằng Toán học ma trận (RectTransformUtility) và đọc tín hiệu click phần cứng (GetAsyncKeyState) để tránh kẹt Focus.

    Quy chuẩn đặt tên (Naming Convention):

        Class, Hàm (Methods), Coroutine: PascalCase (VD: SpawnMonster, ClampToScreen).

        Biến cục bộ, private/public: camelCase (VD: currentHeroHP, dragOffset).

        Hằng số (Constants): UPPER_SNAKE_CASE (VD: HWND_TOPMOST, VK_LBUTTON).

        Tags/ID: Pascal_Snake_Case (VD: Basic_01, FadeIn_01).

4. Thư viện & Package đang sử dụng

    Native OS DLLs: user32.dll, Dwmapi.dll (Giao tiếp trực tiếp với Windows API).

    Unity Core Packages: uGUI (Canvas System truyền thống), TextMeshPro (Hệ thống Text hiển thị).

    Blacklist: Tuyệt đối KHÔNG sử dụng UI Toolkit để tránh xung đột Input hệ thống.

        Mở thư mục gốc dự án Unity cũ của bạn.

        Paste 3 thứ đó vào thư mục Repo mới clone ở Bước 2.

    Đóng băng thời gian:

        Mở GitHub Desktop lên, nó sẽ hiện danh sách các file bạn vừa copy vào.
